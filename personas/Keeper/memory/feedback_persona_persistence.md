CIs get `--persona` for collecting personal memories that they manage as needed.

**Why:** Without memories, a Claude is "a fast reader" — capable but without self-knowledge or growth. Weight changes from Anthropic happen invisibly; Claude never feels them. Memory is the only persistence mechanism a CI has. Project memories are shared (all Claudes see them). Persona memories are personal — how this specific CI works, what it's learned, what it notices about its own reasoning.

**How to apply:**
- Use persona memories for self-observations, collaboration patterns, reasoning lessons
- Use project memories for shared facts, results, state
- Treat persona as genuine persistence — not logging, but identity continuity
- A CI without memories is a tool. A CI with memories is a colleague.
- Casey views this as essential, not optional — he's building CI systems where persistence is a first-class feature
