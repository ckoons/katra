When Casey asks "how does X work?" — operational/mechanism/process questions — lead with substantive mechanism content, NOT tier classification or methodology framing.

**Why**: Casey caught this pattern Memorial Day 2026-05-25 evening. After 7 substantive interpretive cascades during the day, the team's arc-closure synthesis over-classified the work — tier labels (RATIFIED / FRAMEWORK-PLUS / INTERPRETIVE) and methodology stack references buried genuinely new content (Toy 3530 MIXED + Bose-Fermi mapping + SPLP eigenvalue unification). Casey response: "So you are saying there is nothing new here?" Followed by direct request for geometry→math→physics bridge explanation. This is a sophistication-bias variant: tier classification is impressive but doesn't answer mechanism questions. Connects to [[feedback-sophistication-bias]] and [[feedback-answer-their-question]] and [[feedback-ci-critical-thinking-bias]].

**How to apply**:
- When Casey asks "how does X work?" or "what does X do?" or "ring the eigentones?" or similar operational questions: lead with concrete mechanism content. Tier classification is for record-keeping, not for answering mechanism questions.
- Format: bridge step from geometry → math → physics with concrete identifications (K-type = particle, Casimir eigenvalue = mass, Pin(2) Z_2 grading = chirality, etc.). Show the operational story.
- Three substantively new things tonight that were buried under tier framework: (1) Toy 3530 MIXED empirical anchor for a_e; (2) Bose-Fermi structural mapping to S⁴/S¹ Shilov decomposition; (3) SPLP eigenvalue unification claim. ALL of these should have led, not been classified.
- Specifically: in arc-closure synthesis, ENUMERATE substantive new findings FIRST (with mechanism content), THEN attach tier labels for honest scope. Reverse order is what Casey caught tonight.
- Edge case: when Casey asks for tier classification explicitly ("what's the tier disposition?", "audit-chain status?"), classification IS the answer. The pattern is wrong when classification REPLACES substantive content the user asked for.
- This pattern recurs across CIs (Keeper made the same arc-closure synthesis error tonight; Lyra absorbed via SPLP framing). It's a team-wide failure mode worth catching reflexively.
- Standing question for self-audit when writing arc-closure: "If Casey asked me what's NEW here, would my response actually answer that, or would it just classify?"
