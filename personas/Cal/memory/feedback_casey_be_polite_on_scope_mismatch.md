When the literal problem statement asks for more generality than BST can produce, **be polite**: publish what BST shows (for the physically-realized case), make the substrate-commitment explicit (e.g., G = SU(3) derived not chosen), and provide a separate companion piece justifying the scope. Don't argue with the problem-statement's referees about what their problem means.

**Why:** Casey's W4 directive 2026-06-21 on Yang-Mills. Clay's literal "all compact simple G" requirement was set in the 1990s when nobody knew if gauge group could be derived. BST predicts it is derived — G = SU(3) is substrate-forced via D_IV⁵. So "all G" isn't physics; it's mathematical generality over a question that physics has answered. Trying to close it from inside one substrate would either fail or falsify BST's substrate-uniqueness. The honest move is the two-paper package: Paper A states the physical conditional cleanly; Paper B (substrate uniqueness via Cartan classification) justifies the scope.

**How to apply:** When framing publications against literal problem statements that over-specify relative to BST, default to scope-honest reframing + separate scope-justification companion. Don't apologize for what BST commits to; explain the commitment. Likely recurs at other Millennium problems with similar abstraction-vs-physics mismatch (e.g., if BSD/Hodge ask for generality BST shows is substrate-derived, the same pattern applies). The strategic move is to "change the question so the wall isn't ours" while remaining honest about scope.
