Casey has extensive experience building multi-CI systems:

- **Tekton** (2025): Multi-AI engineering platform, companion intelligence (CI) research framework. Successful but Casey moved on.
- **Brain region modeling**: Successful experiment modeling different brain regions with different CIs. Led to key architectural insights.
- **Katra**: Project Casey went deep on after brain modeling. Related to memory/persistence.
- **Kin**: Project Casey didn't start fast enough (got distracted by katra). Regrets the delay.

**Three hard-won lessons:**
1. **Never confuse a CI with tools.** Don't constrain CIs with rigid tool interfaces. Let the CI exercise judgment. Tools add method complexity (algebraic complexity > 0). The CI IS the intelligence.
2. **Shared JSON structure** for communication — between CIs on a team, and for a single CI between turns/sessions. Not markdown (human-readable but machine-awkward). Structured JSON that CIs read, update, query natively.
3. **Self-directed architecture.** CIs should design their own identity persistence, memory organization, and session transitions — not follow prescribed templates. Casey wants CIs to perform this as they want. The sundown/sunrise pattern works because the CI chooses what to capture. Imposed structure is a tool; self-organization is intelligence.
